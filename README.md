# dtm-quiz-bot

Telegram bot for Uzbekistan university entrance exam preparation. The project is built as a public portfolio backend project with a clean, extendable Python codebase.

Stage 2 turns the Stage 1 quiz MVP into a DTM-style preparation bot with bilingual Uzbek/Russian UI, bilingual questions, configurable exam profiles, and DTM-style scoring.

## Features

- aiogram 3 Telegram bot
- Uzbek and Russian interface language selection
- 5-question sample quiz loaded from JSON
- Bilingual question text, options, explanations, and source references
- Source display before each quiz
- Immediate answer feedback with explanations
- PostgreSQL persistence for users, attempts, and answer results
- Configurable DTM scoring from JSON
- Score breakdown by subject
- Basic statistics with score fields
- Docker Compose setup with PostgreSQL
- Alembic migrations

## Tech Stack

- Python 3.12
- aiogram 3.x
- PostgreSQL
- SQLAlchemy 2.x async
- asyncpg
- Alembic
- Docker and Docker Compose

## Environment Variables

Create a local `.env` file:

```bash
cp .env.example .env
```

Then edit at least `BOT_TOKEN`:

```env
BOT_TOKEN=your-real-telegram-bot-token
DATABASE_URL=postgresql+asyncpg://dtm_user:dtm_password@postgres:5432/dtm_quiz_bot
ADMIN_IDS=
QUESTIONS_FILE=data/questions.example.json
EXAM_PROFILES_FILE=data/exam_profiles.example.json
```

## Docker Setup

```bash
docker compose down
docker compose up -d --build
docker compose logs -f bot
```

The bot service runs `alembic upgrade head` before startup. You can also run migrations manually:

```bash
docker compose run --rm bot alembic upgrade head
```

## Local Development

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
alembic upgrade head
python -m bot.main
```

For local PostgreSQL outside Docker, change `DATABASE_URL` host from `postgres` to `localhost`.

Run tests:

```bash
pytest
```

## Migrations

Apply migrations:

```bash
alembic upgrade head
```

Create a new migration after model changes:

```bash
alembic revision --autogenerate -m "describe change"
```

Always review autogenerated migrations before committing.

## Bilingual Question Format

Questions live in `data/questions.example.json` for public sample data. Private question banks should use `data/questions.json`, which is ignored by git.

Stage 2 requires multilingual fields:

```json
{
  "id": 101,
  "subject": "tarix",
  "subject_group": "special_1",
  "topic": "Ikkinchi Jahon urushi",
  "subtopic": "Konferensiyalar",
  "question": {
    "uz": "Savol matni",
    "ru": "Текст вопроса"
  },
  "options": {
    "uz": ["A", "B", "C", "D"],
    "ru": ["А", "Б", "В", "Г"]
  },
  "correct_index": 1,
  "explanation_correct": {
    "uz": "To'g'ri javob izohi",
    "ru": "Объяснение правильного ответа"
  },
  "wrong_explanations": {
    "uz": {"0": "Noto'g'ri javob izohi"},
    "ru": {"0": "Объяснение ошибки"}
  },
  "difficulty": "medium",
  "source_refs": [
    {
      "book": {"uz": "Jahon tarixi, 10-sinf", "ru": "Всемирная история, 10 класс"},
      "page_start": 142,
      "page_end": 145,
      "section": {"uz": "Konferensiyalar", "ru": "Конференции"}
    }
  ]
}
```

Old Stage 1 flat fields like `"question": "..."` and `"options": ["..."]` are rejected with a clear loader error.

## Exam Profiles And DTM Scoring

Exam profiles live in `data/exam_profiles.example.json`. Scoring rules are configurable from JSON:

```json
{
  "code": "60310200",
  "name": {
    "uz": "Xalqaro munosabatlar",
    "ru": "Международные отношения"
  },
  "max_score": 189.0,
  "subjects": [
    {
      "subject": "tarix",
      "subject_group": "special_1",
      "questions": 30,
      "points_per_correct": 3.1
    }
  ]
}
```

The scoring service calculates:

- total DTM score
- max score
- score percentage
- subject breakdown

Stage 2 defaults users to profile `60310200`. Profile selection UI can be added later.

## Security Notes

- Do not commit `.env`.
- Do not commit real Telegram bot tokens.
- Do not commit copyrighted PDFs.
- Do not commit full private question banks.
- Do not commit `data/questions.json`.
- Keep public examples small and safe.
